Actualmente solo hay dos funciones de cara al usuario en raybonsai: generate_tree() y render_tree(). generate_tree()genera un árbol que sigue un cierto conjunto de restricciones que usted establece y devuelve una escena de renderización de rayos que describe el árbol. render_tree() automáticamente agrega terreno, configura la iluminación y configura la cámara para que el árbol esté en el marco, pero por lo demás es solo una envoltura de luz alrededor de la render_scene() función de rayrender .

Probémoslo, con la configuración predeterminada:

library(raybonsai)

generate_tree() %>% 
  render_tree()

¡Un árbol! Vamos a personalizarlo. El crecimiento del árbol se controla principalmente mediante cinco entradas:

  • El número de ramas en cada punto de ramificación: branch_split
  • El ángulo de ramificación horizontal: branch_angle
  • El ángulo de ramificación vertical: branch_angle_vert
  • La longitud escalada de cada rama subsiguiente: branch_scaley (el nombre más apropiado, dado el caso de uso)
  • La semilla aleatoria seed.

Primero, comencemos generando un montón de plantas diferentes, cada una con la misma configuración pero diferentes semillas.

par(mfrow=c(1,2))
for(i in 1:2) {
  generate_tree(seed = i) %>% 
    render_tree()
}

Cada árbol es diferente, pero todos parecen provenir de la misma “especie”. Y eso se debe a que todos se generan utilizando el mismo “ADN”: los ángulos de las ramas y los factores de escala. Cambiemos ese ADN especificando un nuevo conjunto de reglas.

Aquí, cortamos los ángulos potenciales que las ramas pueden formar de -45 y 45 grados a la mitad:

par(mfrow=c(1,2))
for(i in 1:2) {
  generate_tree(seed = i, branch_angle_vert = seq(-45,45,by=5)/2, leaf_color = "pink") %>% 
    render_tree()
}

par(mfrow=c(1,3))
generate_tree(seed = 10, branch_angle = c(-30,0, 30), branch_scale = c(0.6,0.7),
              branch_depth = 7, leaf_color = "green", leaf_depth_start = 5) %>% 
  render_tree()

generate_tree(seed = 11, branch_angle = c(-30,0, 30), branch_scale = c(0.9,1),
              branch_depth = 7, leaf_color = "green", leaf_depth_start = 5) %>% 
  render_tree()

generate_tree(seed = 12, branch_angle = c(-30,0, 30), branch_scale = c(1.1,1.2),
              branch_depth = 7, leaf_color = "green", leaf_depth_start = 5) %>% 
  render_tree()

Figura 3: Variando el ángulo de ramificación vertical. Estos son visualmente bastante distintos del lote anterior y tienen una apariencia similar entre sí. Pero mire de cerca y compare estos árboles con el primer lote: verá que en realidad tienen exactamente la misma estructura de ramificación, solo que con ángulos de ramificación menos pronunciados (tenga en cuenta que los árboles que se inclinan hacia la izquierda / derecha en el primer lote todavía se inclinan de esa manera) el segundo). Esto se debe a que la elección aleatoria de bifurcarse hacia la izquierda o hacia la derecha está controlada por la semilla aleatoria, que es idéntica entre los dos lotes. La única variable que es diferente es el propio ángulo de ramificación.

¿Qué sucede cuando agregamos un ángulo de suma a la mezcla? Aquí, uno directamente en el centro. Ahora, cada rama puede ir a la izquierda, a la derecha o continuar en la misma dirección que la rama de origen. Podemos ver que esto da como resultado que algunos árboles desarrollen “troncos” y algunas ramas rectas más largas:

par(mfrow=c(2,3)) for(i in 1:6) { generate_tree(seed = i, branch_angle_vert = c(-20,0, 20), leaf_color = “red”) %>% render_tree() }

Figura 4: Añadiendo la posibilidad de hacer crecer una rama en la misma dirección que la anterior. La siguiente variable que podemos controlar es aumentar la profundidad de ramificación del árbol. De forma predeterminada, generamos 6 capas de ramas. El número de ramas crece exponencialmente con cada capa, así que simplemente lo aumentaremos a 8. Esto aumentará la complejidad visual de nuestro árbol.

par(mfrow=c(2,3)) for(i in 1:6) { generate_tree(seed = i+1000, branch_depth = 8, leaf_color = “purple”) %>% render_tree() }

Figura 5: Árboles más profundos. De forma predeterminada, raybonsai solo agrega hojas a la última capa, pero podemos comenzar a cultivarlas en capas anteriores para completar nuestro árbol:

par(mfrow=c(1,2))

generate_tree(seed = 1234, branch_depth = 8, leaf_color = “orange”) %>% render_tree()

generate_tree(seed = 1234, branch_depth = 8, leaf_color = “orange”, leaf_depth_start = 5) %>% render_tree()

Figura 6: Relleno de nuestro árbol con hojas adicionales. Hasta este punto, hemos mantenido los árboles simétricos estableciendo un ángulo negativo coincidente para cada ángulo positivo. También podemos hacer árboles asimétricos aumentando la probabilidad de que seleccionemos un ángulo sobre otro. Aquí cada rama tiene una probabilidad 5 veces mayor de girar en una dirección frente a la otra, implementada duplicando un ángulo 5 veces en el vector de ramificación.

par(mfrow=c(1,2)) generate_tree(seed = 2020, branch_angle_vert = c(-10, 10, 10, 10, 10, 10), branch_depth = 8, leaf_color = “magenta”, leaf_depth_start = 5) %>% render_tree() generate_tree(seed = 2021, branch_angle_vert = c(10,-10,-10,-10,-10,-10), branch_depth = 8, leaf_color = “magenta”, leaf_depth_start = 5) %>% render_tree()

Figura 7: Doblado de árboles. También podemos crear árboles asimétricos asegurándonos de que el ángulo promedio sea negativo o positivo, sin repetir ángulos individuales. Esto funciona porque, en promedio, el árbol favorece una dirección sobre la otra.

par(mfrow=c(1,2)) generate_tree(seed = 4321, branch_angle_vert = seq(-10,20,by=1), branch_depth = 8, leaf_color = “yellow”, leaf_depth_start = 5) %>% render_tree()

generate_tree(seed = 4321, branch_angle_vert = -seq(-10,20,by=1), branch_depth = 8, leaf_color = “yellow”, leaf_depth_start = 5) %>% render_tree()

Figura 8: Más flexión de árboles. Otra variable que podemos controlar es el factor de escala branch_scale, que es el predeterminado c(0.8, 0.9)(lo que significa que cada rama tiene un 80% o un 90% de la longitud de la capa anterior). Si disminuimos o aumentamos esto, el árbol resultante será dramáticamente diferente:

par(mfrow=c(1,3)) generate_tree(seed = 10, branch_angle = c(-30,0, 30), branch_scale = c(0.6,0.7), branch_depth = 7, leaf_color = “green”, leaf_depth_start = 5) %>% render_tree()

generate_tree(seed = 11, branch_angle = c(-30,0, 30), branch_scale = c(0.9,1), branch_depth = 7, leaf_color = “green”, leaf_depth_start = 5) %>% render_tree()

generate_tree(seed = 12, branch_angle = c(-30,0, 30), branch_scale = c(1.1,1.2), branch_depth = 7, leaf_color = “green”, leaf_depth_start = 5) %>% render_tree()

Figura 9: Variando el ancho de la rama. Hasta ahora, cada árbol se ha cultivado con el método del “punto medio”: en lugar de hacer crecer cada rama desde el final de la rama anterior, el comportamiento predeterminado extiende la rama en la dirección de las ramas anteriores hasta que alcanza el punto medio, y solo entonces extiende un rama al punto final. Podemos desactivar esta función y configurar midpoint = FALSE: Esto da como resultado un árbol estructuralmente idéntico con una apariencia ligeramente diferente. Y para divertirnos, también juguemos con el color de fondo y el color de la rama:

par(mfrow=c(1,2)) generate_tree(seed = 20, branch_angle = c(-30,0, 30), branch_scale = c(0.9,1), midpoint = TRUE, branch_depth = 7, leaf_color = “chartreuse4”, leaf_depth_start = 5, branch_color = “tan”) %>% render_tree(ground_color1 = “darkgoldenrod4”, ground_color2 = “chocolate4”)

generate_tree(seed = 20, branch_angle = c(-30,0, 30), branch_scale = c(0.9,1), midpoint = FALSE, branch_depth = 7, leaf_color = “chartreuse4”, leaf_depth_start = 5, branch_color = “tan”) %>% render_tree(ground_color1 = “darkgoldenrod4”, ground_color2 = “chocolate4”)

Figura 10: Dos métodos diferentes para cultivar ramas. Izquierda: punto medio. Derecha: no punto medio. La iluminación aquí la configura automáticamente render_tree(), pero podemos apagarla y configurar nuestra propia iluminación usando rayrender. También estableceremos branch_split = 3aquí para un árbol más denso con más ramas:

library(rayrender)

par(mfrow=c(1,1)) generate_tree(seed = 222, branch_angle = c(-20, 20), branch_scale = c(0.8,0.9), branch_split = 3, branch_depth = 6 , leaf_color = “chartreuse4”, leaf_depth_start = 5, branch_color = “tan”) %>% add_object(sphere(x=5,y=1,radius=1,material=light(color=“magenta”,intensity = 30))) %>% add_object(sphere(x=-5,y=1,radius=1,material=light(color=“dodgerblue”,intensity = 30))) %>% raybonsai::render_tree(lights = FALSE, ground_color1 = “grey50”,ground_color2 = “grey50”, width=1200,height=800)

Figure 11: Adding custom lighting to our scene, using rayrender primitives. También podemos cargar una imagen HDR (aquí, obtenida de forma gratuita en hdrihaven.com) para iluminar la escena con una iluminación realista:

generate_tree(seed = 222, branch_angle = c(-20,20), branch_scale = c(0.8,0.9), branch_depth = 10 , leaf_color = “chartreuse4”, leaf_depth_start = 5, branch_color = “tan”) %>% raybonsai::render_tree(lights = FALSE, environment_light = “kiara_3_morning_2k.hdr”, width=1200, height=800)

Y finalmente, podemos crear animaciones como la que se muestra en la parte superior variando las entradas y guardando cada fotograma en una imagen. Una vez renderizados todos los fotogramas, los combinamos con el paquete {av} (contenedor R ffmpeg) en una película:

t_steps = seq(0,360,length.out = 61)[-61] branch_angle1 = 15 * sinpi(t_steps/180) branch_angle2 = 10 * sinpi(t_steps/180+30/180) branch_angle3 = 10 * sinpi(t_steps/180+60/180)

for(i in seq(1,60,by=1)) { generate_tree(seed = 2222, branch_angle_vert = c(-20,20) + branch_angle2[i]/2, branch_angle = seq(-45,45,by=5), branch_depth = 8, leaf_color = “chartreuse4”, leaf_depth_start = 5, branch_color = “tan”) %>% add_object(group_objects(generate_tree(seed = 3333, branch_angle_vert = c(-15,15) + branch_angle1[i]/2, branch_depth = 8 , leaf_color = “dodgerblue4”, leaf_depth_start = 5, branch_color = “tan”), pivot_point = c(0,-10,0),group_angle=c(0,0,-15))) %>% add_object(group_objects(generate_tree(seed = 4444, branch_angle_vert = c(-10,10) + branch_angle3[i]/2, branch_depth = 8, leaf_color = “magenta”, leaf_depth_start = 5, branch_color = “tan”), pivot_point = c(0,-10,0),group_angle=c(0,0,15))) %>% raybonsai::render_tree(lights = FALSE, environment_light = “symmetrical_garden_2k.hdr”, width=1200, height=800, aperture=0.5, filename = glue::glue(“wave{i}”), fov=25,lookfrom=c(0,5,20), lookat=c(0,2,0), rotate_env=-90, sample_method=“stratified”) }
